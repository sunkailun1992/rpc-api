# RPC 契约安全规范

## 字段安全

- 禁止在 DTO 中暴露密码、token、密钥、支付凭证、私钥、短信验证码或第三方 access key。
- 用户、租户、权限、部门、数据范围字段必须只暴露调用方确实需要的最小集合。
- 内部审计字段、风控字段、删除标记、登录失败原因等敏感内部状态默认不跨服务暴露。

## 权限边界

- `rpc-api` 只定义契约，不做鉴权。
- provider 服务必须在实现中完成权限、租户、数据范围、脱敏和幂等校验。
- consumer 不能因为 RPC 返回了数据就跳过自身业务场景的权限判断。

## 上下文安全

- 登录用户、租户、数据源、请求 ID、Seata XID、流量泳道和版本上下文由受控过滤器透传。
- 不允许业务方随意伪造高权限用户上下文。
- 日志中只记录必要 ID，不输出 token、密钥或大段请求体。

## 变更检查

- 新增 DTO 字段时检查是否含敏感信息。
- 新增接口时检查是否会绕过 provider 原有 HTTP 权限逻辑。
- 修改租户或数据源相关契约时，必须编译 provider/consumer 并说明租户隔离验证方式。
